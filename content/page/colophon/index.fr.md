Ce site Web, [harper.blog](https://harper.blog), est le blog personnel de Harper Reed. Il a été construit à l’aide de technologies Web modernes et de techniques de génération de sites statiques.

## Stack technique

- **Générateur de site statique** : [Hugo](https://gohugo.io/)  
- **Hébergement** : [Netlify](https://www.netlify.com/)  
- **Contrôle de version** : Git (hébergé sur GitHub)

## Conception visuelle et mise en page

- Le site utilise un thème personnalisé basé sur le thème [Bear Cub](https://github.com/clente/hugo-bearcub) ᕦʕ •ᴥ•ʔᕤ  
- ʕ•̫͡•ʕ•̫͡•ʔ•̫͡•ʔ•̫͡•ʕ•̫͡•ʔ•̫͡•ʕ•̫͡•ʕ•̫͡•ʔ•̫͡•ʔ•̫͡•！  
- Typographie : des polices système sont utilisées pour des performances optimales et un rendu natif  
- Un design responsive garantit la compatibilité avec différents appareils et tailles d’écran

## Gestion du contenu

- Le contenu est rédigé au format Markdown

## Génération et déploiement

- Le déploiement continu est assuré via Netlify  
- Le site est automatiquement généré et déployé lorsqu’un changement est poussé sur la branche principale  
- Des commandes et paramètres personnalisés sont définis dans le fichier `netlify.toml`

## Optimisation des performances

- Les images sont compressées et servies au format WebP lorsque c’est possible  
- Le CSS est minifié pour les versions de production  
- Le pipeline d’assets intégré d’Hugo est utilisé pour optimiser les ressources

## Fonctionnalités supplémentaires

- Un flux RSS est disponible pour la syndication du contenu  
- Des balises méta pour les réseaux sociaux facilitent le partage sur des plateformes telles que Twitter et Facebook  
- Des shortcodes personnalisés sont employés pour enrichir le formatage du contenu (p. ex. intégration Kit.co)

## Outils de développement

- Un Makefile simplifie les tâches de développement courantes  
- Le projet gère ses dépendances via Go modules

## Accessibilité et standards

- Le site vise l’accessibilité et respecte les standards Web modernes  
- Du HTML sémantique est utilisé sur l’ensemble du site

## Analytics

- Le site utilise [tinylytics](https://tinylytics.app/) pour suivre divers indicateurs (« bits ») ainsi que les visites (« hits »). Vous pouvez consulter les résultats [ici](https://tinylytics.app/public/cw1YY9KSGSE4XkEeXej7).
