Diese Website, [harper.blog](https://harper.blog), ist der persönliche Blog von Harper Reed. Sie wurde mit modernen Webtechnologien entwickelt und nutzt Techniken zur statischen Seitengenerierung.

## Tech-Stack

- **Statischer Seitengenerator**: [Hugo](https://gohugo.io/)
- **Hosting**: [Netlify](https://www.netlify.com/)
- **Versionsverwaltung**: Git (gehostet auf GitHub)

## Design und Layout

- Die Seite verwendet ein individuelles Theme, das auf dem [Bear Cub](https://github.com/clente/hugo-bearcub)-Theme basiert. ᕦʕ •ᴥ•ʔᕤ

- Typografie: Systemschriften sorgen für optimale Performance und eine systemeigene Optik
- Responsive Design gewährleistet eine optimale Darstellung auf allen Geräten und Bildschirmgrößen

## Content Management

- Die Inhalte werden in Markdown verfasst

## Build und Deployment

- Kontinuierliche Bereitstellung (Continuous Deployment) ist über Netlify eingerichtet
- Die Seite wird automatisch neu gebaut und veröffentlicht, sobald Änderungen in den Branch ›main‹ übertragen werden
- Benutzerdefinierte Build-Befehle und Einstellungen sind in `netlify.toml` definiert

## Performance-Optimierungen

- Bilder werden optimiert und – wenn möglich – im WebP-Format ausgeliefert
- CSS wird für Produktions-Builds minifiziert
- Die integrierte Asset-Pipeline von Hugo optimiert die Ressourcen

## Weitere Funktionen

- Ein RSS-Feed steht zur Verfügung, um Inhalte zu abonnieren oder weiterzuverbreiten
- Social-Media-Meta-Tags verbessern das Teilen auf Plattformen wie Twitter und Facebook
- Eigene »Shortcodes« ermöglichen eine erweiterte Inhaltsformatierung (z. B. Kit.co-Integration)

## Entwicklungstools

- Ein `Makefile` automatisiert gängige Entwicklungsaufgaben
- Das Projekt verwendet Go-Module für die Abhängigkeitsverwaltung

## Barrierefreiheit und Standards

- Die Website strebt Barrierefreiheit an und hält sich an moderne Webstandards
- Dabei kommt durchgängig semantisches HTML zum Einsatz

## Analytics
